# The Continuity Tool

A simple prototype for exploring how systems determine whether a new entry should be treated as:

* a new record
* a duplicate
* a related or supplemental record

## Problem

Many workflows rely on manual review, spreadsheets, and repetitive checks to figure out whether new information belongs to an existing record or should be treated as new.

That creates:

* duplicate work
* inconsistent decisions
* routing errors
* wasted time

## Purpose

This prototype demonstrates a simple continuity check process that compares a new entry against existing records and suggests whether it is likely:

* New
* Duplicate
* Related / Supplemental

## What it demonstrates

* workflow integrity thinking
* record matching logic
* decision support for repetitive manual processes
* system design focused on accuracy and consistency
